FEATURE REQUIREMENTS:

1️⃣ Navbar (Persistent)
- Title: **Events Around**
- Links: “Search” and “Favorites”
- Icons: Search + Heart
- Responsive: collapses into hamburger menu on mobile

2️⃣ Search Route `/search`
Form fields (5 total):
1. **Keyword** — text input with Ticketmaster autosuggest (spinner while loading, “×” to clear)
2. **Category** — dropdown: All, Music, Sports, Arts & Theatre, Film, Miscellaneous
3. **Location** — text input; disabled when “Auto-detect location” checked
4. **Auto-detect location** — toggle switch; calls `ipinfo.io`
5. **Distance** — numeric field with “miles” suffix  
Buttons:
- “Search Events” (black button)
- “Clear” (resets all fields)

Behaviors:
- Input validation → red border + helper text (see Figure 8)
- Auto-detect disables Location and fills with geolocation
- Location manually entered → geocoded using Google Maps API
- Debounce autosuggest (to reduce API calls)
- Send GET request to backend `/api/searchEvents?...`

3️⃣ Results Grid
- Display up to 20 events from Ticketmaster JSON
- Each card shows:
  - Category badge
  - Date/time
  - Event image
  - Event name
  - Venue name
  - Favorite icon (heart)
- Sort ascending by date/time
- Click card → `/event/:id`
- Empty state → “Nothing found” (see no-search-result.png)

4️⃣ Event Details Page `/event/:id`
Top bar:
- “Back to Search” link → restores search and scroll
- Title, “Buy Tickets” button (Ticketmaster URL)
- Favorite icon (toggles + toast)

Tabs:
1. **Info Tab**
   - Date, Artists/Teams, Venue, Genres, Price Range, Ticket Status (color-coded)
   - Seat Map
   - Facebook & X (Twitter) icons → open share links
2. **Artist Tab**
   - Only visible for “Music” genre
   - Use Spotify API: followers, popularity, Spotify link, top albums
3. **Venue Tab**
   - Name, Address (Google Maps link), See Events button, Image
   - Parking, General Rule, Child Rule

Ticket Status color codes:
- On Sale: Green
- Off Sale: Red
- Postponed: Orange
- Rescheduled: Orange
- Canceled: Black

5️⃣ Favorites Page `/favorites`
- Display cards identical to search results
- Sorted by time added
- Persist in MongoDB Atlas (ReadWrite user)
- Empty state: “No favorite events yet”
- Notifications:
  - “Added to favorites!”
  - “Removed from favorites!” (includes Undo)
  - “Re-added to favorites!”

Use **Sonner** for stackable toasts with hover unstack animation.

6️⃣ Backend API Endpoints
Use Express.js routes:
```

GET    /api/suggest?keyword=
GET    /api/searchEvents?keyword=&radius=&unit=&geoPoint=&segmentId=
GET    /api/eventDetails/:id
GET    /api/favorites
POST   /api/favorites/add
DELETE /api/favorites/remove/:id

```
Notes:
- Only backend can call Ticketmaster API.
- Frontend can directly call ipinfo.io and Google Geocoding API.
- Include MongoDB connection (`mongoose` or `mongodb` driver).
